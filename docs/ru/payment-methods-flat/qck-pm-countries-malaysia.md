## Особенности проведения платежей

`Shopee`При проведении платежей с использованием платежного методаQR-код для оплаты действителен 5 минут. Если пользователь не выполнит необходимые для оплаты действия в течение этого времени, платеж отклоняется.

`Touch&amp;Go`Особенности проведения платежей с использованием платежного метода:

- `Touch&amp;Go`Выплата с использованием методадоступна только для мерчантов категории Financial Services (Forex, Binaries) по предварительному согласованию с курирующим менеджером Some Platform.
- Если пользователь не выполнит необходимые для оплаты действия в течение 15 минут после инициирования оплаты, платеж отклоняется.
- Если пользователь при оплате укажет неверную сумму, платеж останется в промежуточном статусе, пока конфликт не будет разрешен вручную техническими специалистами Some Platform.
- Если в обработке уже находится такое количество платежей, что в указанном диапазоне не удается найти свободного значения, то этот и все последующие запросы отклоняются платежной платформой, пока не завершится обработка хотя бы одного из текущих платежей с той же суммой.Если указанное увеличение суммы платежа произойдет, то в оповещении вашей системе будет передана измененная сумма.Вот пример поведения платежной платформы при последовательном поступлении двух запросов с одинаковой суммой:В платежную платформу поступает первый запрос на оплату 15,00 MYR. Так как это первый такой платеж, то платформа начинает выполнение платежа с суммой 15,00 MYR.В платежную платформу поступает второй запрос на оплату 15,00 MYR и к этому моменту проведение первого платежа не закончено.Так как платеж на сумму 15,00 MYR уже выполняется, платежная платформа изменяет сумму второго платежа на 15,01 MYR и начинает выполнять его именно с этой суммой.Если сумма в запросе на платеж совпадает с суммой одного из текущих платежей, то из-за особенностей работы провайдера платежная платформа может (но не обязательно делает это) увеличить сумму платежа на величину от 0,01 до 0,99 MYR. Это иногда требуется, чтобы измененная сумма была уникальной среди других находящихся в обработке платежей с той же суммой. Платформа избегает увеличения суммы платежа, если это возможно.
