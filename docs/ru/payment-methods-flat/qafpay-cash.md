## Пополнение лицевого счета через агента provider_service

### Общая информация

Чтобы выполнить оплату через Gate с использованием метода title:

Полная схема оплаты с использованием метода title выглядит следующим образом.

1. Примите и обработайте запрос на проверку существования в вашей системе лицевого счета пользователя.
2. Отправьте ответ с информацией о существовании лицевого счета пользователя в вашей системе.
3. Примите и обработайте запрос на получение идентификатора платежа в вашей системе.
4. Отправьте ответ с идентификатором платежа.
5. Примите оповещение о результате пополнения лицевого счета пользователя.

### Запрос

1. `verify`Чтобы проверить, существует ли в вашей системе лицевой счет пользователя, платежная платформа Some Platform отправляет POST-запросна URL-адрес вашей системы.
2. type`verify`— тип операции в вашей системе, значение параметра всегда;customer_id— идентификатор учетной записи пользователя в вашей системе, на баланс которой зачисляются денежные средства;project_id— идентификатор проекта, полученный от Some Platform при интеграции;signature— подпись запроса.Пример данных из запроса на проверку существования лицевого счета пользователя{
    "type": "verify",
    "project_id": 1234,
    "customer_id": "7123456789",
    "signature": "e19695449545fac562843ca06e7e0f3e51281ad6"
}Запрос содержит следующие обязательные параметры:
3. `verify`code— код ответа на запрос;currency`0`— код валюты платежа в формате ISO-4217 alpha-3. Данный параметр обязателен для ответа с кодом.additional_customer_id— идентификатор пользователя, уникальный в рамках проекта, который используется для идентификации пользователя в платежной платформе Some Platform;message— произвольное описание кода ответа.Допустимые коды ответовКодОписание`0`Операция успешна`400`Ошибка подписи`404`Счет пользователя не существует`500`messageОбщая ошибка, причина должна быть описана в параметреОтвет может содержать следующие необязательные параметры:`0`В случае успеха проверки наличия в вашей системе лицевого счета ответ должен содержать кодответа на запрос и код валюты платежа.`404message`Если запрос нельзя корректно обработать или не удалось обнаружить запрошенный лицевой счет, ответ на запрос должен содержать код ответа, а также может содержать дополнительную информацию об ошибке в параметре.Пример ответа на запрос{

redirect_dataЧтобы перенаправить пользователя, используйте значения следующих параметров, находящихся в объекте:

Вот пример фрагмента оповещения, содержащего данные для перенаправления пользователя:

- url— адрес для перенаправления пользователя;
- body— данные для отправки в теле запроса;
- method`POSTGET`— метод запроса HTML-страницы сайта (например,или);
- encrypted— это служебный параметр. Игнорируйте содержащиеся в нем данные.

```json
"redirect_data": {
    "method": "GET",
    "body": [],
    "encrypted": [],
    "url": "https://example.com/redirect"
}
```

Вот пример фрагмента оповещения, содержащего данные для перенаправления пользователя:

- url— адрес для перенаправления пользователя;

}
