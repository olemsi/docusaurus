# Тестирование

По умолчанию при активации плагина используется тестовый режим. Этот режим позволяет ознакомиться с работой плагина на сайте, к которому он подключен, без проведения реальных операций. В тестовом режиме можно настраивать и проверять работу плагина, проводить тестовые заказы и просматривать информацию по ним.

## Настройка

Для настройки параметров работы плагина в тестовом режиме необходимо:

У плагина есть следующие параметры:

- Поле— для отображения метода оплаты с помощью платежной страницы Some Platform на сайте.**Status**
- Параметры работы платежной страницы Some Platform при ее отображении:
- Другие параметры работы плагина:
- Параметрыидля подключения к платежным решениям Some Platform.**Project IDSecret key**

1. Перейти на вкладку.
2. Выбрать в списке cms_paymentmethod и щелкнуть кнопку.**Edit**
3. Задать необходимые параметры (см. далее).
4. Убедиться, что в полевыбрано.**Test modeTrue**
5. Щелкнуть кнопку.**Save**

## Проведение тестовых заказов

После настройки параметров работы плагина следует провести тестовые заказы на сайте и проверить информацию по этим заказам в cms_oscommerce, на вкладке.**Orders**

::: tip
В тестовом режиме для изменения статуса заказа снапосле совершения платежа необходимо щелкнуть кнопку. В противном случае заказ остается со статусом.**PendingProcessingВернуться на сайтPending**
:::
