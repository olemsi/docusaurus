# Оплата через Gate. Канал 2: с перенаправлением пользователя

## Общая информация

Для оплаты через Gate с использованием метода title со стороны веб-сервиса необходимо:

Полная схема проведения оплаты выглядит следующим образом.

ru_gate_sts_protocol.ditaИнформация о формате запросов и параметрах инициации оплат методом title через Gate, а также о формате оповещений о результатах оплат приведена далее, общая информация о работе с API см. в разделе.

1. Отправить запрос, содержащий требуемые параметры и подпись, на рабочий URL Some Platform.
2. Осуществить перенаправление в сервис title.
3. Принять оповещение от платежной платформы Some Platform о результате оплаты.

## Формат запросов

При работе с запросами на оплаты с использованием метода title необходимо учитывать следующее:

Вот пример данных из запроса на оплату с использованием метода title:

1. `/v2/payment/banks/vietnam-qr/sale`Должен использоваться POST-запрос к конечной точке. Эта конечная точка относится к группе конечных точек request-payment-banks-sale.
2. generalproject_id— идентификатор проекта, полученный от Some Platform при интеграции;payment_id— идентификатор платежа, уникальный в рамках проекта;signatureИспользование подписи к данным— подпись запроса, составленная после указания целевых параметров (подробнее — в разделе);— объект, содержащий основные сведения о запросе:customerid— идентификатор пользователя, уникальный в рамках проекта;ip_address— IP-адрес устройства пользователя;first_name— имя пользователя;last_name— фамилия пользователя;email— адрес электронной почты пользователя;— объект, содержащий сведения о пользователе:paymentcurrency— код валюты платежа в формате ISO-4217 alpha-3;amount*в дробных единицах валюты без десятичной точки и пробелов*ru_currency_codes.ditaЕсли у валюты есть дробные единицы (то есть количество разрядов дробных единиц не равно нулю), то в этом параметре нужно указывать сумму в. Подробнее о разрядах дробных единиц у валют см..— сумма оплаты в основных единицах валюты, без дробной части.— объект, содержащий сведения о платеже:accountbank_idПараметр может быть необязательным, для уточнения обратитесь к курирующему менеджеру Some Platform— идентификатор банка..— сведения о счете пользователя:return_urlsuccessПараметр может быть необязательным, для уточнения обратитесь к курирующему менеджеру Some Platform— URL для перенаправления при успешной оплате..— объект, содержащий URL для перенаправления пользователя в веб-сервис:В запросе должны использоваться следующие объекты и параметры:
3. return_urlreturn_urldecline— URL-адрес, на который нужно перенаправить пользователя в случае отклонения оплаты;returnsuccessdecline— URL-адрес, куда нужно перенаправить пользователя в случае прерывания оплаты пользователем до ее завершения. Этот же адрес используется, если не заданы параметрыи.— объект с информацией, куда следует перенаправлять пользователя при прерывании или завершении оплаты:Следующие параметры с адресами возврата пользователя необязательны, но их настоятельно рекомендуется передавать в запросе, чтобы сделать процесс оплаты максимально удобным для пользователя. Если не задан ни один из параметров объекта, то Gate по умолчанию перенаправляет пользователя по URL-адресу, указанному в вашем проекте в Some Platform.
4. Дополнительно могут использоваться любые другие параметры, указанные в спецификации.

## Формат данных для перенаправления пользователей

После получения и обработки запроса на оплату платежная платформа направляет в ваш веб-сервис оповещение (callback) с данными для перенаправления пользователя в сервис provider_service.

redirect_dataЧтобы перенаправить пользователя, используйте значения следующих параметров, находящихся в объекте:

Вот пример фрагмента оповещения, содержащего данные для перенаправления пользователя.

- url— адрес для перенаправления пользователя;
- body— данные для отправки в теле запроса;
- method— метод запроса HTML-страницы сайта (например, POST или GET);
- encrypted— это служебный параметр. Игнорируйте содержащиеся в нем данные.
