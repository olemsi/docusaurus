### Общая информация

Для оплаты через Gate с использованием метода title со стороны веб-сервиса необходимо:

Полная схема проведения оплаты представлена далее.

ru_gate_sts_protocol.ditaИнформация о формате запросов и параметрах инициализации оплат методом title через Gate, а также о формате оповещений о результатах оплат приведена далее, общая информация о работе с API см. в разделе.

1. Отправить запрос, содержащий требуемые параметры и подпись, на рабочий URL Some Platform.
2. Осуществить перенаправление пользователя на сайт сервиса title.
3. Принять оповещение о результате оплаты.

### Формат запросов

При работе с запросами на оплаты с применением метода title необходимо учитывать следующее:

Вот пример данных из запроса на оплату с использованием метода title:

1. `/v2/payment/cash/cashtocode/sale`Должен использоваться запрос, отправляемый методом POST. Этот запрос относится к группе запросов для получения оплат от пользователей через title терминалы: request-payment-cash-sale.
2. generalproject_id— идентификатор проекта, полученный от Some Platform при интеграции;payment_id— идентификатор платежа, уникальный в рамках проекта;signatureИспользование подписи к данным— подпись запроса, составленная после определения всех параметров запроса. Подробнее о разрядах дробных единиц у валют см..— сумма оплаты в дробных единицах валюты без десятичной точки и пробелов в соответствии с приведенной выше.currency— код валюты оплаты в формате ISO-4217 alpha-3.— объект, содержащий сведения об оплате:В запросе должны использоваться следующие объекты и параметры:
3. Могут указываться только те валюты, которые поддерживаются для данного метода с учетом региональных ограничений.
4. return_urlreturn_urlsuccess— URL-адрес, на который нужно перенаправить пользователя в случае успешного завершения оплаты;decline— URL-адрес, на который нужно перенаправить пользователя в случае отклонения оплаты;returnsuccessdecline— URL-адрес, куда нужно перенаправить пользователя в случае прерывания оплаты пользователем до ее завершения. Этот же адрес используется, если не заданы параметрыи.— объект с информацией, куда следует перенаправлять пользователя при прерывании или завершении оплаты:Следующие параметры с адресами возврата пользователя необязательны, но их настоятельно рекомендуется передавать в запросе, чтобы сделать процесс оплаты максимально удобным для пользователя. Если не задан ни один из параметров объекта, то Gate по умолчанию перенаправляет пользователя по URL-адресу, указанному в вашем проекте в Some Platform.
5. Дополнительно могут использоваться другие параметры, указанные в спецификации.

### Форматы данных для перенаправления пользователей

После получения и обработки запроса на оплату платежная платформа направляет в ваш веб-сервис оповещение (callback) с данными для перенаправления пользователя в сервис title.

redirect_dataЧтобы перенаправить пользователя, используйте значения следующих параметров, находящихся в объекте:

Вот пример фрагмента оповещения, содержащего данные для перенаправления пользователя.

- url— адрес для перенаправления пользователя;
- body— данные для отправки в теле запроса;
- method— метод запроса HTML-страницы сайта (например, POST или GET);
- encrypted— это служебный параметр. Игнорируйте содержащиеся в нем данные.

6. ru_pp_parameters.ditaДополнительно могут использоваться любые другие параметры из числа доступных для работы с Payment Widget. Детальная информация обо всех параметрах приведена в разделе.
7. ru_pp_authentication.ditaПосле определения всех параметров необходимо составить подпись. Подробную информацию см. в.
