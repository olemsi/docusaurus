# Ответ

## Общая информация

*ответа*Платежная платформа подтверждает получение запроса отправкой— HTTP-сообщения заданной структуры — в течение того же сеанса. Передаваемые в ответе данные содержат:

в спецификации интерфейсаДалее представлена информация об общей структуре ответов, а также о кодах, используемых для передачи информации о состоянии запроса. Описание структур данных для ответов на конкретные запросы представлено.

- запрошенную информацию, если запрос успешно выполнен;
- сведения об ошибке, если запрос выполнить нельзя.

## Структура

В каждом ответе от платформы содержатся следующие элементы в порядке перечисления:

- `HTTP/1.1200 OK`стартовая строка с указанием протокола и его версии (), кода ответа и поясняющей фразы к коду (например,);
- поля заголовка;
- пустая строка — разделитель, отделяющая служебную информацию от тела сообщения;
- тело сообщения, содержащее JSON-строку в кодировке UTF-8 в с набором данных.

## Коды ответа

HTTP-код ответа используется в стартовой строке каждого ответа для передачи информации о результате выполнения запроса либо о причине обнаруженной ошибки. В работе с statistics_api применяются следующие коды ответов и пояснительные фразы.

| Код с пояснением | Описание |
| --- | --- |
| 200 OK | Запрос успешно выполнен. В теле ответа передана запрошенная информация |
| 400 Bad Request | Запрос не может быть принят из-за синтаксической ошибки, обнаруженной при извлечении данных из JSON-строки, или из-за отсутствия в наборе извлеченных данных обязательных параметров (кроме токена и подписи) |
| 401 Authorization Required | Запрос не может быть принят из-за отказа в доступе, например если в запросе некорректно переданы токен или подпись либо указан идентификатор проекта, к которому нет доступа через используемый токен |
| 403 Forbidden | Запрос не может быть принят из-за отсутствия у используемой учетной записи прав доступа к соответствующему разделу Dashboard |
| 500 Internal Error | Запрос не может быть выполнен из-за сбоя в платежной платформе |

- `nameAuthorization Required`поясняющая фраза к коду из заголовка ответа в параметре(в примере —);
- `messageYou have no access to project_id = 22`описание ошибки в параметре(если ошибка была обработана в платформе; в примере —);
- `code0`служебный код в параметрес фиксированным значением;
- `status400`код из заголовка ответа в параметре(в примере —).
