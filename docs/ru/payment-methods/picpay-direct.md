## Оплата через Gate

### Общая информация

При выполнении оплаты через Gate с использованием метода title веб-сервис должен выполнить следующее:

Полная схема оплаты выглядит следующим образом.

ru_gate_sts_protocol.ditaИнформация о формате запросов и параметрах инициирования оплаты методом title через Gate, а также о формате оповещений о результатах оплаты приведена далее. Общую информацию о работе с Gate API можно найти в разделе.

1. Отправить запрос с нужными параметрами и подписью на рабочий URL-адрес Some Platform.
2. Выполнить перенаправление в сервис provider_service.
3. Принять от платежной платформы Some Platform оповещение о результате оплаты.

### Формат запросов

При работе с запросами на оплату с использованием метода title необходимо учитывать следующее:

Вот пример данных из запроса на оплату с использованием метода title:

1. Следует использовать POST-запрос в конечную точку.
2. generalproject_id— идентификатор проекта, полученный от Some Platform при интеграции;payment_id— идентификатор платежа, уникальный в рамках проекта;signatureИспользование подписи к данным— подпись запроса, составленная после определения всех параметров запроса. (Подробнее см. раздел.);— объект, содержащий основные сведения о запросе:customerid— идентификатор пользователя, уникальный в рамках проекта;ip_address— IP-адрес устройства пользователя;email— адрес электронной почты пользователя;first_name— имя пользователя;last_name— фамилия пользователя;— объект, содержащий сведения о пользователе:paymentamount*за исключениемосновных единицах валюты*ru_currency_codes.dita— сумма оплаты в дробных единицах валюты без десятичной точки и пробеловслучаев, когда у валюты нет дробной части. Если у валюты нет дробных единиц (то есть количество разрядов дробных единиц равно нулю), то в этом параметре нужно указывать сумму в. Подробнее о разрядах дробных единиц у валют см.;currency— код валюты оплаты в формате ISO-4217 alpha-3.— объект, содержащий сведения об оплате:В запросе должны присутствовать следующие объекты и параметры:
3. customerphone— номер телефона пользователя, начинается на 55, не более 13 цифр;identifydoc_number— номер CPF пользователя (номер налогоплательщика в Бразилии, 11 цифр).— объект, содержащий сведения о документе, подтверждающем личность:— объект, содержащий сведения о пользователе:ru_gate_clarification.ditaЕсли не передать эти параметры в исходном запросе на оплату, то платежная платформа направит в веб-сервис оповещение со списком недостающих параметров. (Подробнее о процедуре уточнения параметров запроса см..) Если передать в исходном запросе все необходимые параметры, то процедура уточнения параметров запроса не выполняется.Также необходимо передавать следующие параметры:
4. return_urlreturn_urlsuccess— URL-адрес, на который нужно перенаправить пользователя в случае успешного завершения оплаты;decline— URL-адрес, на который нужно перенаправить пользователя в случае отклонения оплаты;returnsuccessdecline— URL-адрес, куда нужно перенаправить пользователя в случае прерывания оплаты пользователем до ее завершения. Этот же адрес используется, если не заданы параметрыи.— объект с информацией, куда следует перенаправлять пользователя при прерывании или завершении оплаты:Следующие параметры с адресами возврата пользователя необязательны, но их настоятельно рекомендуется передавать в запросе, чтобы сделать процесс оплаты максимально удобным для пользователя. Если не задан ни один из параметров объекта, то Gate по умолчанию перенаправляет пользователя по URL-адресу, указанному в вашем проекте в Some Platform.
5. Дополнительно в запросе могут использоваться любые другие (необязательные) объекты и параметры, указанные в спецификации.

### Формат данных для перенаправления пользователей

redirect_dataЧтобы перенаправить пользователя, используйте значения следующих параметров, находящихся в объекте:

- encrypted— это служебный параметр. Игнорируйте содержащиеся в нем данные.

## Оплата через Payment Widget

### Общая информация

При выполнении оплаты через Payment Widget с использованием метода title веб-сервис отправляет запрос, содержащий требуемые параметры и подпись, на рабочий URL-адрес Some Platform и принимает оповещение о результате оплаты.

ru_pp__preselectingps.ditaПри этом существует возможность открыть Payment Widget так, чтобы на ней уже был выбран метод title. (Подробнее о предварительном выборе метода см..)

Полная схема проведения оплаты выглядит следующим образом.

ru_pp_api.ditaИнформация о формате запросов и параметрах вызова Payment Widget при работе с методом title, а также о формате оповещений о результатах оплаты приведена далее. Общую информацию о работе с Payment Widget API можно найти в разделе.

### Формат запросов

При создании запросов на открытие платежной формы с использованием метода title необходимо учитывать следующее:

Вот пример данных из запроса на открытие платежной формы с использованием метода title:

1. project_id— идентификатор проекта, полученный от Some Platform при интеграции;payment_id— идентификатор платежа, уникальный в рамках проекта;customer_id— идентификатор пользователя, уникальный в рамках проекта;customer_first_name— имя пользователя;customer_last_name— фамилия пользователя;customer_email— адрес электронной почты пользователя;payment_currency— код валюты платежа в формате ISO-4217 alpha-3;payment_amount*за исключениемосновных единицах валюты*ru_currency_codes.dita— сумма платежа в дробных единицах валюты без десятичной точки и пробеловслучаев, когда у валюты нет дробной части. Если у валюты нет дробных единиц (то есть количество разрядов дробных единиц равно нулю), то в этом параметре нужно указывать сумму в. Подробнее о разрядах дробных единиц у валют см..Должен использоваться базовый минимум параметров, которые обязательны для любого платежного метода:
2. customer_phone— номер телефона пользователя, начинается на 55, не более 13 цифр;identify_doc_number— номер CPF пользователя (номер налогоплательщика в Бразилии, 11 цифр).ru_pp_clarification.ditaЕсли какой-либо из этих параметров не передать в запросе на открытие платежной формы, то на Payment Widget пользователю будет предложено указать соответствующую информацию. Подробнее о процедуре уточнения параметров см..Дополнительно нужно передавать следующие параметры с данными пользователя:
3. force_payment_method`force_pm`Если нужно, чтобы Payment Widget открылась с уже выбранным методом title, добавьте в запрос параметрсо значением.
4. ru_pp_parameters.ditaДополнительно можно использовать любые другие параметры из числа доступных для работы с Payment Widget. Подробнее о возможных параметрах Payment Widget см..
5. ru_pp_authentication.ditaПосле определения всех параметров запроса необходимо составить подпись. (Подробнее о создании подписи см..)

### Формат оповещений

`1234example_amount_in_majors&#160;example_currency`В следующем примере в оповещении содержится информация о том, что в проектевыполнена оплата в размере.
