Reference

API references for payments and reporting

Use the transactional API when your backend creates payments and handles callbacks. Use the data API when you need balances, operations, and reporting data.

Transactional

Host-to-host API

Direct server-to-server API for payment creation, refunds, payouts, verification, and status checks.

`POST` + JSON body
Signed payloads with `project_id` and `secret_key`
Synchronous responses plus backend callbacks
Open reference
Reporting

Data API

Retrieval API for balances, operations, payments, and other reporting-oriented data from the platform.

`POST` + JSON body
Access with `token` and request `signature`
Built for reporting, reconciliation, and analytics access
