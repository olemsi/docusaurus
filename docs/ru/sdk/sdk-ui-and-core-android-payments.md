# Проведение платежей

`Sale`По умолчанию в sdk_ui_android настроено проведение разовых одностадийных оплат (с типом действия). Для проведения оплат такого типа можно использовать приведенные выше примеры и дополнительно ничего не настраивать.

Вместе с тем при работе с sdk_ui_android можно проводить и двухстадийные оплаты (с блокировкой средств через SDK и последующим списанием). Для этого следует:

1. `sdk_ui_android_host_first_capActionType.AuthpaymentOptions`sdk_ui_android_host_first_capPaymentOptions.sdk_ui_android_host_first_capActionType.Auth;Вызвать платежную форму, задав тип действияв объекте:
2. ru_db_payments_purchases.ditaКогда потребуется, подтвердить списание средств через Dashboard (подробнее см.) или через Gate (с помощью запроса к конечной точке).
