::: tip
When selecting projects or terminals is not needed, the corresponding drop-down lists are not shown in the designer interface. At the same time, certain terminals can be unavailable in the designer, and certain terminals with custom interface models can be available to be selected, but unavailable to be changed. If you have any questions related to projects and terminals, contact the Platform technical support.
:::

## Adding a style

To add a design style:

1. **Style**Select the style whose settings need to be inherited in thedrop-down list.
2. Clickon the right of the selected style's name.
3. Name the new style in the dialogue box that opens.
4. **Continue**Clickto confirm the creation of the new style.
5. **Style**Make sure that the designer interface shows the page of the created style and its name is specified in thedrop-down list.

## Applying the style

To ensure that a certain style is used for the specific project or terminal, you should:

1. If necessary, select the project, terminal, and style from the corresponding drop-down lists.
2. **Continue**Clickat the bottom of the tools panel.
3. **Save &amp; Set Active**Clickto apply the style.
4. Make sure you can see the notification that the style has been applied.

::: tip
**Style**When you need to apply one style to several terminals, repeat these steps for each terminal. If you select a terminal, and the required style is not available in thedrop-down list, make sure that the same interface model is used for this terminal as for other terminals.
:::
