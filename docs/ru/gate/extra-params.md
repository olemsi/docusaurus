# Использование дополнительных параметров проведения платежей

## Информация о мерчанте

merchant.descriptorrequest-payment-card-sale — запрос на оплату в одну стадию,request-payment-card-sale-saved — запрос на оплату по сохраненным данным карты,request-payment-card-sale-token — запрос на оплату по токену карты,request-payment-card-refund — запрос на возврат,request-payment-card-recurring — запрос на проведение регулярного платежа.При необходимости можно передавать краткую информацию, идентифицирующую мерчанта, который осуществил операцию. Эта информация отражается в банковской выписке держателя карты и может содержать название мерчанта или его проекта, адрес, номер телефона или адрес электронной почты. Необходимые данные передаются в параметрев одном из запросов, поддерживающих данную функциональность:

`# $ * ( ) - _ = + " ; : ? \ / , . ` ' ^`Максимальное количество передаваемых символов в параметре — 13,  допустимо использование букв латинского алфавита, цифр и следующих символов:.

О возможности использования функциональности уточняйте у своего курирующего менеджера.

## Условия проведения оплаты

payment.extra_paramGate позволяет задавать особые условия обработки платежей. Для этого необходимо передать данные в параметре.

Дополнительные сведения об этой возможности уточняйте у своего курирующего менеджера.
