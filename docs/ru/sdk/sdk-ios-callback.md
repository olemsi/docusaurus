# Обработка оповещений

## Обзор

При работе с sdk_ios поддерживается использование оповещений, отправляемых платежной платформой Some Platform напрямую к веб-сервису мерчанта, и оповещений, передаваемых вначале от платежной формы к sdk_ios, а затем — от sdk_ios к мобильному приложению мерчанта. В первом случае набор параметров, передаваемых в оповещении, может отличаться в зависимости от установленных настроек (подробнее:). Во втором случае передается строго определенный набор параметров — данные о статусе, типе и идентификаторе платежа, дате проведения платежа,сумме и валюте платежа:используемом платежном методе,ru_pp_callbacks.dita#ru_pp_callbacks/section_zlt_rgh_4bc

Далее представлена информация об этих оповещениях.

```
{
    "payment": {
        "status": "success",
        "type": "purchase",
        "id": "12345",
        "date": "2020-09-11T14:49:18+0000",
        "method": "card",
        "sum": 1000,
        "currency": "USD"
         }
}
```

## Оповещения платежной платформы

При проведении платежа платежная платформа отправляет оповещения на указанный URL. Этот URL необходимо сообщить службе технической поддержки Some Platform. Подробная информация об оповещениях представлена в разделе.ru_pp_callbacks.dita

## Оповещения sdk_ios

Мобильное приложение мерчанта может принимать от sdk_ios оповещения, которые передаются к sdk_ios от платежной платформы. Оповещение отправляется мерчанту до того, как пользователю в платежной форме отображается страница с информацией о результате проведения платежа.

Для получения оповещения с информацией о результате проведения платежа необходимо использовать публичный протокол. В рамкахзадействуется метод, который выполняется автоматически при получении финального статуса платежа и при выполнении которого предоставляется информация о платеже. Для получения информации о результате проведения платежа можно использовать приведенные далее примеры и дополнительно ничего не настраивать; при необходимости также возможно изменение мерчантом данного кода.`sdk_ios_callbacksdk_ios_callback`
