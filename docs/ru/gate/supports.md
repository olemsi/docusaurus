# Обзор изменений

*базовойрасширенной*Для поддержки 3ds2 в платежной платформе вводится ряд изменений, предполагающих использование одной из двух схем аутентификации:, основанной на уже существующей схеме аутентификации 3ds1, но включающей в себя промежуточные перенаправления пользователя, или, позволяющей избежать промежуточных перенаправлений, но требующей доработок на стороне веб-сервиса. Общая информация обо всех изменениях, а также о действиях, необходимых со стороны мерчантов для поддержки 3ds2, представлена в этом разделе.

| СХЕМА | ИЗМЕНЕНИЯ | ДЕЙСТВИЯ |
| --- | --- | --- |
| Базовая | request-payment-card-sale,request-payment-card-sale-saved,request-payment-card-sale-token,request-payment-card-auth,request-payment-card-auth-saved,request-payment-card-auth-token,request-payment-card-account-verification.Новые объекты и параметры не обязательны, но их использование рекомендуется для того, чтобы повысить вероятность выбора варианта аутентификации frictionless flow.Добавлены объекты и параметры для запросов к следующим конечным точкам:Расширена структура оповещения о результате проведения платежа. В оповещениях о результатах тех платежей, при проведении которых выполнялась аутентификация 3ds2, передаются новые параметры.Поддержан новый алгоритм перенаправления пользователя в процессе аутентификации: с формированием HTML-страницы на стороне веб-сервиса и перенаправлением пользователя на страницу ожидания платежной платформы | Требуется заменить логотипы программ аутентификации международных платежных систем: Mastercard Secure Code на Mastercard Identity Check и Verified by Visa на Visa Secure.Если на стороне веб-сервиса поддерживался протокол 3ds1, то допустимо не дорабатывать программный код, но для повышения вероятности выбора варианта аутентификации frictionless flow рекомендуется выполнить доработки с учетом изменений в платежной платформе |

В оповещениях о результатах тех платежей, при проведении которых выполнялась аутентификация 3ds2, передаются новые параметры.Поддержан новый алгоритм перенаправления пользователя в процессе аутентификации: с открытием на стороне веб-сервиса объекта iframe и перенаправлением пользователя на страницу аутентификации.`3ds_notification_url`Добавлена конечная точка request-payment-card-3ds_check_iframe. Запросы к этой точке необходимы только в тех случаях, когда в запросах на проведение платежей передается параметрс адресом для получения уведомления о приеме данных об устройстве пользователя. Использование такой возможности задается со стороны мерчанта.`cres`Добавлен параметрдля указания информации о результате аутентификации 3ds2 в запросах на продолжение платежей к конечной точке request-payment-card-3ds_result.Изменена структура оповещений с данными для перенаправления пользователя | Требуется заменить логотипы программ аутентификации международных платежных систем: Mastercard Secure Code на Mastercard Identity Check и Verified by Visa на Visa Secure.Необходимо доработать программный код с учетом изменений в платежной платформе |
