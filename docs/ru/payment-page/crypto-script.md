# Скрипт шифрования карточных данных

## Общая информация

Платежная платформа Some Platform позволяет проводить оплаты по картам и проверки действительности карты с использованием платежной формы вашего веб-сервиса. Такой способ помогает избежать дополнительных перенаправлений пользователей и самостоятельно настраивать маршрутизацию платежей. Для настройки функциональности необходимо установить скрипт шифрования в исходный код платежной страницы вашего веб-сервиса, который собирает необходимые параметры, шифрует данные карты и создает токен для этой карты.

[лист самооценки](https://www.pcisecuritystandards.org/pci_security/completing_self_assessment)Для проведения платежей с использованием платежных карт необходимо предоставить курирующему менеджеру Some Platform документ о соответствии требованиям PCI DSS. Это может быть копия сертификата соответствия (при его наличии) или. С вопросами о подключении функциональности и заполнении листа самооценки можно обращаться к курирующему менеджеру Some Platform.

Функциональность доступна в режимах работы Payment Widget: purchase и Verify_Card. Для проведения оплаты или проверки действительности карты с помощью такого токена необходимо отобразить пользователю платежную форму с обязательными для заполнения полями, затем получить ответ от платежной платформы со сгенерированным токеном карты с помощью скрипта, отправить запрос на проведение оплаты или проверки с помощью токена в платежную платформу и получить оповещение о результате оплаты. Полный сценарий проведения оплаты представлен далее.
