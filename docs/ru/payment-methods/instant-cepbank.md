## Оплата через Gate

### Общая информация

Для оплаты через Gate с использованием метода title со стороны веб-сервиса необходимо:

Полная схема проведения оплаты представлена далее.

ru_gate_sts_protocol.ditaИнформация о формате запросов и параметрах инициирования оплат методом title через Gate, а также о формате оповещений о результатах оплат приведена далее, общая информация о работе с API представлена в разделе.

1. Отправить запрос, содержащий требуемые параметры и подпись, на рабочий URL Some Platform.
2. Осуществить перенаправление в сервис title.
3. Принять оповещение от платежной платформы Some Platform о результате оплаты.

### Формат запросов

При работе с запросами на оплаты с применением метода title необходимо учитывать следующее:

Вот пример данных из запроса на оплату с использованием метода title:

1. `/v2/payment/bank-transfer/instant-cepbank/sale`Должен использоваться запрос, отправляемый методом POST. Этот запрос относится к группе запросов на оплату с помощью банковского перевода: request-payment-bank-transfer-sale.
2. generalproject_id— идентификатор проекта;payment_id— идентификатор платежа;signatureИспользование подписи к данным— подпись запроса, составленная после указания целевых параметров (подробнее — в разделе);— объект, содержащий основные идентификационные сведения запроса:paymentamount*за исключениемосновных единицах валюты*ru_currency_codes.dita— сумма оплаты в дробных единицах валюты без десятичной точки и пробеловслучаев, когда у валюты нет дробной части. Если у валюты нет дробных единиц (то есть количество разрядов дробных единиц равно нулю), то в этом параметре нужно указывать сумму в. Подробнее о разрядах дробных единиц у валют см.;currency— валюта платежа в формате ISO-4217 alpha-3;— объект, содержащий сведения о платеже:customerid— идентификатор, уникальный в рамках проекта,ip_address— IP-адрес устройства пользователя;first_name— имя пользователя;last_name— фамилия пользователя;email— адрес электронной почты пользователя;country— страна в формате ISO 3166-1 alpha-2;day_of_birth— дата рождения в формате ДД-ММ-ГГГГ;— объект, содержащий сведения о пользователе:return_urlsuccess— URL-адрес, на который нужно перенаправить пользователя в случае успешного завершения оплаты;decline— URL-адрес, на который нужно перенаправить пользователя в случае отклонения оплаты.— объект с информацией, куда следует перенаправлять пользователя при прерывании или завершении оплаты:В запросе должны использоваться следующие объекты и параметры:
3. return_urlreturnsuccessdecline— URL-адрес, куда нужно перенаправить пользователя в случае прерывания оплаты пользователем до ее завершения. Этот же адрес используется, если не заданы параметрыи.— объект с информацией, куда следует перенаправлять пользователя при прерывании или завершении оплаты:Следующий параметр с адресом возврата пользователя необязателен, но его настоятельно рекомендуется передавать в запросе, чтобы сделать процесс оплаты максимально удобным для пользователя.
4. references/ru/currencies/TRY.ditaВалютой платежа может быть только.
5. Дополнительно могут использоваться любые другие параметры, указанные в спецификации.

### Данные для перенаправления пользователей

После получения и обработки запроса на оплату платежная платформа направляет в ваш веб-сервис оповещение (callback) с данными для перенаправления пользователя в сервис provider_service.

redirect_dataЧтобы перенаправить пользователя, используйте значения следующих параметров, находящихся в объекте:

Вот пример фрагмента оповещения, содержащего данные для перенаправления пользователя.

- url— адрес для перенаправления пользователя;
- body— данные для отправки в теле запроса;
- method— метод запроса HTML-страницы сайта (например, POST или GET);
- encrypted— это служебный параметр. Игнорируйте содержащиеся в нем данные.

```json
"redirect_data": {
    "method": "GET",
    "body": [],
    "encrypted": [],
    "url": "https://example.com/redirect"
}
```

3. references/ru/currencies/TRY.ditaВалютой платежа может быть только.
4. force_payment_method`force_pm`Для предварительного выбора метода title необходимо указывать код платежного метода в параметре—.
5. ru_pp_parameters.ditaДополнительно могут использоваться любые другие параметры из числа доступных для работы с Payment Widget. Детальная информация обо всех параметрах приведена в разделе.
6. ru_pp_authentication.ditaПосле определения всех параметров необходимо составить подпись. Подробная информация представлена в разделе.
