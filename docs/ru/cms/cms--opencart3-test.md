# Тестирование

По умолчанию при активации плагина используется тестовый режим. Этот режим позволяет ознакомиться с работой плагина на сайте, к которому он подключен, без проведения реальных операций. В тестовом режиме можно настраивать и проверять работу плагина, проводить тестовые заказы и просматривать информацию по ним.

## Настройка

Для настройки параметров работы плагина в тестовом режиме необходимо:

На вкладкенастраиваются основные параметры работы плагина:**Settings**

На вкладкеможно присвоить названия статусам заказа, описанным в пункте.**Order Status**Схема работы

- . Определяет режим отображения метода оплаты с помощью платежной страницы Some Platform на сайте.**Status**
- Параметры работы платежной страницы Some Platform при ее отображении:
- . Определяет регионы, для которых доступен данный платежный метод.**Geo Zone**
- и. Используются для подключения к платежным решениям Some Platform.**Project IDSecret key**

1. Перейти на вкладкуи выбрать тип расширения—.**Оплата**<!-- image -->
2. Найти в таблицестроку со способом оплаты cms_paymentmethod и щелкнуть расположенную в этой строке кнопку.**Оплата**<!-- image -->
3. Щелкнуть в этой же строке кнопку, чтобы настроить параметры работы плагина.<!-- image -->
4. Задать необходимые настройки (см. далее).
5. Убедиться, что установлен флажок.**Test mode**
6. Щелкнуть кнопку, чтобы сохранить изменения.<!-- image -->

## Проведение тестовых заказов

После настройки параметров работы плагина следует провести тестовые заказы на сайте и проверить информацию по этим заказам в cms_opencart30, на вкладке.

::: tip
В тестовом режиме для изменения статуса заказа снапосле совершения платежа необходимо щелкнуть кнопку. В противном случае заказ остается со статусом.**Pending statusSuccess statusВернуться на сайтPending status**
:::
