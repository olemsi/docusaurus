# Фильтрация платежных методов

## Общая информация

При работе с Payment Widget поддерживается возможность скрывать от пользователя платежные методы, доступные в рамках используемого проекта, но неактуальные для конкретного платежа (с учeтом специфики региона, пользователя или иных факторов). При использовании этой возможности на странице выбора платежного метода пользователю отображаются все доступные методы, за исключением скрытых.

`hide`в соответствующем разделеЧтобы скрыть определенные методы в рамках конкретного сеанса работы Payment Widget, в запросе на открытие Payment Widget необходимо передать параметрс кодами этих методов, разделив их запятой. Коды поддерживаемых платежных методов представлены. Никаких дополнительных действий для подключения и использования этой возможности не требуется.

В следующем примере из числа доступных для проведения платежа исключаются методы UPI и Skrill.

```json
{
   "project_id": 43,
   "payment_id": "456790",
   "payment_currency": "USD",
   "payment_amount": 131970,
   "customer_id": "customer_12", 
   "hide": "upi, skrill-wallet",   // коды скрываемых платежных методов
   "signature": "TSzdE5rJZaA9TYAKoGpfXriFf82MxF..."
 }
```

::: tip
подробнееЕсли для проведения платежа актуален только один платежный метод, рекомендуется использовать возможность предварительного выбора метода ().
:::

## Дополнительные материалы

При работе с возможностью фильтрации платежных методов могут быть полезны следующие материалы:

- ru_pp_force_payment_method_codes.dita— справочный раздел с кодами поддерживаемых платежных методов.
- ru_pp__preselectingps.dita— раздел с информацией о выборе конкретного метода для проведения платежа.
- ru_pm_intro.dita— раздел с информацией о платежных методах и работе с ними.
- ru_pp_parameters.dita— раздел с описанием параметров вызова Payment Widget.
