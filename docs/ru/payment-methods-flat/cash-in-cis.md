# Обзор

title — платежный метод для проведения оплат с использованием физических терминалов на территории Содружества Независимых Государств (СНГ).

references/ru/currencies/USD.ditareferences/ru/currencies/USD.dita* Пользователь может оплатить с использованием любой валюты, которая доступна в выбранном терминале. На стороне провайдера происходит конвертация валюты платежа в, и все платежи на стороне платежной платформе и веб-сервиса проходят в валюте.

В проведении отдельного платежа с использованием title задействуются веб-сервис мерчанта, один из интерфейсов и платежная платформа Some Platform, а также технические средства провайдера.

## Оплата через Gate

### Общая информация

Для оплаты с использованием метода title со стороны веб-сервиса необходимо:

Полная схема проведения оплаты представлена далее.

Информация о формате запросов и параметрах инициирования оплат через Gate при работе с методом title, а также о формате оповещений о результатах оплат приведена далее.

1. Обработать запрос на проверку существования лицевого счета пользователя.
2. Отправить ответ с информацией о существовании лицевого счета пользователя.
3. Обработать запрос на получение идентификатора платежа на стороне веб-сервиса.
4. Отправить ответ с идентификатором платежа.
5. Принять оповещение о результате оплаты.

### Формат запросов

**`verify`Запросна проверку существования лицевого счета пользователя**

**`check_deposit`Запросна получение идентификатора платежа**

`check_deposit`Запросдля получения идентификатора платежа на стороне веб-сервиса отправляется методом POST на заданный URL веб-сервиса.type`check_deposit`— тип операции на стороне веб-сервиса, значение параметра всегда;customer_id— идентификатор пользователя в системе мерчанта;project_id— идентификатор проекта, полученный от Some Platform при интеграции;amount*за исключениемосновных единицах валюты*ru_currency_codes.dita— сумма оплаты в дробных единицах валюты без десятичной точки и пробеловслучаев, когда у валюты нет дробной части. Если у валюты нет дробных единиц (то есть количество разрядов дробных единиц равно нулю), то в этом параметре нужно указывать сумму в. Подробнее о разрядах дробных единиц у валют см.;currency— валюта платежа в формате ISO-4217 alpha-3;payment_method`allpay-kz`для Allpay,`briogroup-uz`для BRIO Group,`fenix-kz`для Fenix,`indigo24-kz`для Indigo24,`moneymovers-ge`для Money Movers,`quickpay-kg`для Quickpay,`w1-kz`для Wallet One.— код терминальной сети. Допустимые значения:signature— подпись запроса.Пример запроса на получение идентификатора платежа на стороне веб-сервиса{
   "type":"check_deposit",
   "project_id":111,
   "customer_id":"johndoe@example.com",
   "amount":10000,
   "currency":"USD",
   "payment_method":"briogroup-uz",
   "signature":"lY0LTSAzpR7zGce5qfYGacOuYlHGWqkMcQlqmjlsDDZI2gV
                     cE1qVeWANnkIR7mdOqRXJnL1kO0lUmkQ0YYLWRg=="
}Запрос содержит следующие обязательные параметры:

1. `verify`Запросдля проверки существования лицевого счета пользователя отправляется методом POST на заданный URL веб-сервиса.
2. type`verify`— тип операции на стороне веб-сервиса, значение параметра всегда;customer_id— идентификатор пользователя в системе мерчанта;project_id— идентификатор проекта, полученный от Some Platform при интеграции;signature— подпись запроса.Пример запроса на проверку существования лицевого счета пользователя{
   "type":"verify",
   "project_id":111,
   "customer_id":"johndoe@example.com",
   "signature":"lY0LTSAz85R7zGce5qfYGacOuYlHGWqkMcQlqmjlsDDZI2gVcE1q
                              VeWANnkIR7mdOqRXJnL1kO0lUmkQ0YYLWRg=="
}Запрос содержит следующие обязательные параметры:

### Формат ответов

`verify`Ответ на запросдолжен быть отправлен синхронно и содержать следующие обязательные параметры:

`check_deposit`Ответ на запросдолжен быть отправлен синхронно и содержать следующие обязательные параметры:

| Код | Описание |
| --- | --- |
| `0` | Операция успешна |
| `400` | Ошибка подписи |
| `404` | Лицевой счет пользователя не существует |
| `500` | errors.messageОбщая ошибка, причина должна быть описана в необязательном параметре |

### Формат оповещений

ru_gate_callbacks.ditaДля оповещений о результатах оплат с применением метода title используется стандартный формат, описание которого представлено в разделе.

`methodpayment`В параметреобъектапередается название терминальной сети с использованием которой была проведена оплата.

`238100,01 USDBrio Group`В следующем примере оповещение свидетельствует о том, что в рамках проектабыла успешно проведена оплата в размерес использованием терминальной сети.
