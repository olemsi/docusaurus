# Основные возможности и ролевая модель

Dashboard представляет собой единый интерфейс для специалистов мерчанта. В нем поддерживается полный набор требуемых функций, и для того, чтобы распределять доступ к разным функциям между разными специалистами (с учетом организационной структуры и прочих факторов), в Dashboard используется ролевая модель.

В этом разделе описаны ключевые возможности, поддерживаемые роли и нюансы их использования, а также распределение прав между ролями.

## Возможности

Контроль и анализ проведения платежей — с поддержкой просмотра, фильтрации, поиска и визуализации информации о проводимых и проведенных платежах.Выполнение финансовых операций — с поддержкой оплат разных типов, выплат и возвратов (как полных, так и частичных).Ведение финансового учета — с поддержкой подготовки и просмотра различных отчетов и с доступом к информации о балансах.Администрирование работы с интерфейсом и проектами — с поддержкой управления учетными записями пользователей, включая распределение ролей и контроль действий, и с поддержкой просмотра свойств доступных проектов, включая настройку URL-адресов для возврата пользователей в проект после проведения платежей.Возможности Dashboard можно разбить на следующие группы:

## Роли

В следующей таблице перечислены роли учетных записей в Dashboard; для каждой роли указаны ее права, а также специальности, которым обычно назначают эти роли.

**Финансы**Набор ролей фиксирован, и у мерчанта нет возможности создавать другие роли. Но при этом для отдельных учетных записей можно корректировать набор прав, заданных через роль, а также назначать учетной записи несколько ролей одновременно и тем самым сочетать права, изначально не входящие в один из базовых наборов. Например, если для учетной записи с ролью Operations требуется доступ к просмотру балансов, то дополнительно можно отнести эту учетную запись к роли Finance с правом просматривать балансы в разделе.

Учетные записи с ролью Merchant Admin наделены полным набором прав и доступом ко всем проектам мерчанта, и такие учетные записи могут создаваться только специалистами технической поддержи Some Platform. Учетные записи с остальными ролями создаются и настраиваются (в том числе в плане доступа к проектам) сотрудниками мерчанта с ролью Merchant Admin. При этом все права, назначенные учетной записи, актуальны для всех проектов, к которым предоставлен доступ для этой учетной записи. Выборочное применение прав в зависимости от проектов в рамках одной учетной записи не используется.
