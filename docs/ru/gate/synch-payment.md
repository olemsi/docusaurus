# Синхронная обработка  платежей

ru_gate_interaction_organisation.ditaВ общем случае синхронная схема обработки в Some Platform используется для сравнительно простых запросов, которые можно полностью выполнить на стороне платежной платформы, например вернуть статус платежа в платформе. Для обработки запросов на платежи такая схема не применяется, потому что они выполняются только асинхронно, как описано в разделе.

**синхронную обработку платежей**Однако в платежной платформе Some Platform существует возможность (по согласованию с курирующим менеджером) подключить, когда платеж выполняется в рамках одного HTTP-сеанса и результат обработки запроса (или даже всего платежа) возвращается в синхронном ответе.

Синхронная обработка платежей выполняется в той же последовательности и на тех же условиях, что описаны в статьях этого сайта, посвященных асинхронной обработке платежей. Разница лишь состоит в том, что после получения запроса на платеж платежная платформа не возвращает ответ о приеме запроса в асинхронную обработку, а начинает синхронно выполнять запрос.

`domen_synch`Запросы на синхронную обработку платежей нужно отправлять по адресу.Доменное имя Gate узнавайте у своего курирующего менеджера или в службе поддержки.Платежная платформа принимает запросы по протоколам HTTP версии не ниже 1.1 и TLS версии не ниже 1.2. Полная спецификация интерфейса доступна по адресу domen_contract_URL.
