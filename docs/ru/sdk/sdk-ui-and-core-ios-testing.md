# Тестирование

При необходимости платежную форму можно открыть в тестовом режиме, чтобы получить информацию об ошибках, допущенных при указании параметров платежа, а при отсутствии ошибок — протестировать проведение оплат с определенным результатом. Для этого в запросе на открытие платежной формы в объектеможно передать следующие значения для параметра(значения указаны для языков Swift и Objective-C соответственно):`PaymentOptionsmockModeType`

Если необходимо открыть платежную форму в рабочем режиме, в параметреследует передать значение/.`mockModeTypeMockModeType.disabledMockModeTypeDisabled`

Также проведение платежей можно протестировать в тестовой среде платежной платформы Some Platform. Для этого подключитесь к тестовой среде Some Platform и укажите полученные идентификатор тестового проекта и соответствующий ему секретный ключ в значении обязательных параметров запроса на открытие платежной формы.

В исходных примерах кода, предоставленных на Github, заданы постоянные значения для этих параметров.

Если вы готовы перейти в рабочий режим, замените тестовые значения — идентификатор рабочего проекта и секретный ключ от него — на рабочие.

- /— если необходим результат «платеж проведен»;`MockModeType.successMockModeTypeSuccess`
- /— если необходим результат «платеж отклонен».`MockModeType.declineMockModeTypeDecline`
