# Выплата на виртуальную карту

## Общая информация

В данном разделе описана выплата через Gate на виртуальную карту.

Пользователь получает выплату на свою виртуальную карту. Если у пользователя нет виртуальной карты, но он является клиентом выбранного банка, платежная платформа запрашивает у банка выпуск новой карты и пользователь получает выплату на эту карту. В финальном оповещении (callback) банк передает в вашу систему информацию о результате выплаты и данные карты в замаскированном виде.

Если пользователь не является клиентом выбранного банка, вам нужно перенаправить пользователя в сервис банка для прохождения предварительной KYC-проверки. После успешной проверки банк выпускает для пользователя новую карту и пользователь получает выплату на эту карту. В финальном оповещении банк передает в вашу систему информацию о результате выплаты и данные карты в замаскированном виде.

## Перенаправление пользователя для выполнения KYC-проверки

Если пользователь не является клиентом банка, с помощью которого проводится выплата, то сервис банка проводит KYC-проверку. Для этого платежная платформа направляет вам оповещение с данными для перенаправления пользователя в сервис банка.

redirect_dataЧтобы перенаправить пользователя, используйте значения следующих параметров, находящихся в объекте:

Вот пример фрагмента оповещения, содержащего данные для перенаправления пользователя:

- url— адрес для перенаправления пользователя;
- body— данные для отправки в теле запроса;
- method`POSTGET`— метод запроса HTML-страницы сайта (например,или);
- encrypted— это служебный параметр. Игнорируйте содержащиеся в нем данные;
- kyc_id— идентификатор, полученный от банка.

```json
"redirect_data": {
    "method": "GET",
    "kyc_id": "state_identifier_12345",
    "body": [],
    "encrypted": [],
    "url": "https://example.com/redirect"
}
```

```
https://example.com/return_url?code=12345&amp;state=state_identifier_12345
```
