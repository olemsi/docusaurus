## Оплата через Gate

### Общая информация

Для оплаты через Gate с использованием метода title со стороны веб-сервиса необходимо:

Полная схема проведения оплаты представлена далее.

ru_gate_sts_protocol.ditaИнформация о формате запросов и параметрах инициирования оплат через Gate при работе с методом title, а также о форматах данных для перенаправления пользователей и о формате оповещений о результатах оплат приведена далее; общая информация о работе с API — в разделе.

1. Отправить запрос, содержащий требуемые параметры и подпись, на рабочий URL Some Platform.
2. Осуществить перенаправление пользователя в сервис title.
3. Принять оповещение о результате оплаты.

### Формат запросов

При формировании запросов на оплату с применением метода title необходимо учитывать следующее:

Вот пример данных из запроса на оплату с использованием метода title:

1. `/v2/payment/online-banking/bankrbk24/sale`Должен использоваться запрос, отправляемый методом POST. Этот запрос относится к группе запросов через интернет-банкинг request-payment-online-banking-sale.
2. generalproject_id— идентификатор проекта, полученный от Some Platform при интеграции;payment_id— идентификатор платежа, уникальный в рамках проекта;signatureИспользование подписи к данным— подпись запроса, составленная после определения всех параметров запроса. (Подробнее см. раздел.);— объект, содержащий основные сведения о запросе:customerid— идентификатор пользователя, уникальный в рамках проекта;ip_address— IP-адрес устройства пользователя;phone— номер телефона пользователя;email— адрес электронной почты пользователя;first_name— имя пользователя;last_name— фамилия пользователя;— объект, содержащий сведения о пользователе:paymentamount*за исключениемосновных единицах валюты*ru_currency_codes.dita— сумма оплаты в дробных единицах валюты без десятичной точки и пробеловслучаев, когда у валюты нет дробной части. Если у валюты нет дробных единиц (то есть количество разрядов дробных единиц равно нулю), то в этом параметре нужно указывать сумму в. Подробнее о разрядах дробных единиц у валют см.;currency— код валюты оплаты в формате ISO-4217 alpha-3.— объект, содержащий сведения об оплате:В запросе должны присутствовать следующие объекты и параметры:
3. return_urlreturn_urlsuccess— URL-адрес, на который нужно перенаправить пользователя в случае успешного завершения оплаты;decline— URL-адрес, на который нужно перенаправить пользователя в случае отклонения оплаты;returnsuccessdecline— URL-адрес, куда нужно перенаправить пользователя в случае прерывания оплаты пользователем до ее завершения. Этот же адрес используется, если не заданы параметрыи.— объект с информацией, куда следует перенаправлять пользователя при прерывании или завершении оплаты:Следующие параметры с адресами возврата пользователя необязательны, но их настоятельно рекомендуется передавать в запросе, чтобы сделать процесс оплаты максимально удобным для пользователя. Если не задан ни один из параметров объекта, то Gate по умолчанию перенаправляет пользователя по URL-адресу, указанному в вашем проекте в Some Platform.
4. references/ru/currencies/EUR.ditareferences/ru/currencies/KZT.ditareferences/ru/currencies/RUB.ditareferences/ru/currencies/USD.ditaВалютой платежа могут быть,,,.
5. Дополнительно в запросе могут использоваться любые другие (необязательные) объекты и параметры, указанные в спецификации.

### Данные для перенаправления пользователей

После получения и обработки запроса на оплату платежная платформа направляет в ваш веб-сервис оповещение (callback) с данными для перенаправления пользователя в сервис provider_service.

redirect_dataЧтобы перенаправить пользователя, используйте значения следующих параметров, находящихся в объекте:

Вот пример фрагмента оповещения, содержащего данные для перенаправления пользователя.

- url— адрес для перенаправления пользователя;
- bodylifetime– время в секундах, в течение которого платеж должен быть завершен;id– идентификатор платежа на стороне сервиса title;— данные для отправки в теле запроса:
- method— метод запроса HTML-страницы сайта (например, POST или GET);
- encrypted— это служебный параметр. Игнорируйте содержащиеся в нем данные.

```json
"redirect_data": {
    "method": "POST",
    "body": {
        "lifetime": 86400,
        "id": 50416966
    },
    "encrypted": [],
    "url": "https://example.com/redirect"
}
```

`23810 000,00 KZT`В следующем примере оповещение свидетельствует о том, что в рамках проектаот пользователя была успешно проведена оплата в размере.

В следующем примере оплата была отклонена, так как время выполнения оплаты истекло.
