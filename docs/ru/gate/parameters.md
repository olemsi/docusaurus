# Формат запроса на платеж через Gate

## Общая информация

При переходе к 3ds2 для запросов на проведение платежей используется стандартный формат с поддержкой параметров, которые не являются обязательными, но рекомендуются для повышения вероятности выбора варианта аутентификации frictionless flow.

ru_gate_interaction_organisation.ditaОбщее описание форматов запросов при работе через Gate представлено в разделе, а описание объектов и параметров, использование которых рекомендуется для повышения вероятности выбора варианта аутентификации frictionless flow, — в этом разделе.

## Объект acs_return_url

Объект содержит параметры, необходимые для перенаправления пользователя к веб-сервису после выполнения аутентификации, а также для получения уведомления о приеме данных от сервера управления доступом (ACS) эмитента при использовании расширенной схемы аутентификации.

| ПАРАМЕТР | ТИП | ОПИСАНИЕ |
| --- | --- | --- |
| `return_url` | type_string | Адрес веб-сервиса для перенаправления пользователя после аутентификации, является обязательным при использовании расширенной схемы |
| `3ds_notification_url` | type_string | Адрес веб-сервиса для получения уведомления о том, что данные приняты сервером управления доступом. Является обязательным, если на стороне мерчанта необходимо получать такие уведомления в рамках аутентификации по расширенной схеме |

## Форматы запросов на проведение платежей через SDK для мобильных приложений

### Общая информация

Выполнение аутентификации 3ds2 поддерживается во всех версиях SDK для мобильных приложений (sdk_android и sdk_ios), однако, начиная с версии 1.5 набора средств разработки sdk_android и версии 1.5.1 набора средств разработки sdk_ios, также поддерживается возможность использования дополнительных параметров, необходимых для повышения вероятности выбора варианта аутентификации frictionless flow. В этом разделе представлена информация об этих параметрах.
