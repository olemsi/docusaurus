ru_pp__preselectingps.ditaПри этом существует возможность открыть Payment Widget так, чтобы на ней уже был выбран метод title. (Подробнее о предварительном выборе метода см..)

ru_pp_api.ditaИнформация о формате запросов и параметрах вызова Payment Widget при работе с методом title, а также о формате оповещений о результатах оплаты приведена далее. Общую информацию о работе с Payment Widget API можно найти в разделе.

Подробнее о разрядах дробных единиц у валют см..terminal_id— уникальный идентификатор шаблона Payment Widget. Уточняйте у вашего курирующего менеджера в Some Platform, какое значение передавать в этом параметре.Должен использоваться базовый минимум параметров, которые обязательны для любого платежного метода:
2. force_payment_method`force_pm`Если нужно, чтобы Payment Widget открылась с уже выбранным методом title, добавьте в запрос параметрсо значением.
3. ru_pp_parameters.ditaДополнительно можно использовать любые другие параметры из числа доступных для работы с Payment Widget. Подробнее о возможных параметрах Payment Widget см..
4. ru_pp_authentication.ditaПосле определения всех параметров запроса необходимо составить подпись. (Подробнее о создании подписи см..)
