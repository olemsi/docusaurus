# Общая информация

## Введение

В этой статье представлена информация об использовании плагина cms_nopcommerce_plugin для различных веб-сервисов, разработанных на базе CMS cms_nopcommerce версий cms_nopcommerce_version и выше.

С помощью описываемого плагина можно проводить оплату и выполнять частичный или полный возвраты с использованием различных платежных методов из числа поддерживаемых для платежной формы Payment Widget. (Подробнее о платежных методах см..). Также при работе с плагином можно использовать различные дополнительные возможности, обеспечиваемые со стороны Some Platform для Payment Widget () и со стороны разработчиков CMS cms_nopcommerce для этой системы. Например, можно поддерживать для пользователейи регулировать. Применяя возможности такого рода, можно подстраиваться под различные региональные особенности, улучшать пользовательские сценарии, повышать конверсию платежной формы и проходимость платежей.подробнееповторные попытки оплатыприменение плагина в разных странах

Для подключения возможностей, предоставляемых Some Platform, следует обращаться к технической документации на этом портале и, по мере необходимости, к специалистам Some Platform.

## Схема работы

В схеме проведения оплат с использованием плагина cms_nopcommerce_plugin задействуются пользователь, веб-сервис со встроенным в него плагином, платежная форма Payment Widget, платежная платформа и платежная среда. При этом на стороне веб-сервиса мерчанта обеспечиваются автоматический вызов Payment Widget и автоматический прием оповещений от платежной платформы в соответствии с параметрами работы плагина.

Каждому заказу, сформированному в веб-сервисе, соответствует оплата с одним из возможных статусов:,,(),,,— в соответствии со статусами платежей в CMS cms_nopcommerce. Информация об управлении заказами через интерфейс cms_nopcommerce представлена.`PendingAuthorizedPaidcapturedPartially refundedRefundedVoided`[в документации к CMS](https://docs.nopcommerce.com/en/running-your-store/order-management/orders.html)

Для начала работы с плагином cms_nopcommerce_plugin его необходимо скачать в интернет-магазине и установить одним из следующих способов:

После установки плагина можно открыть его карточку и настроить параметры его работы. При этом рекомендуется сначала выполнить тестирование с тестовыми параметрами и уже после тестирования задать рабочие параметры и перейти к полноценному использованию плагина.

Чтобы открыть карточку плагина в cms_nopcommerce, перейдите в раздел, выберитеи щелкните кнопкув строке с названием плагина. В карточке плагина cms_nopcommerce_plugin отобразится краткая инструкция по работе с ним и поля для настройки параметров его работы. Описания параметров для тестирования и работы представлены в следующих разделах этой статьи.**ConfigurationPayment methodsConfigure**
