# Открытие Payment Widget на нескольких устройствах

## Общая информация

При использовании Payment Widget можно отображать на платежной форме QR-код для открытия формы на другом устройстве пользователя без прерывания сеанса работы. Это может быть актуально, например, при первоначальном отображении платежной формы на экране телевизора, в то время как пользователю может быть удобнее продолжить оплату на мобильном устройстве.

QR-код может отображаться на следующих страницах платежной формы: странице выбора платежного метода, странице указания платежных данных (за исключением ряда платежных методов), странице выбора платежных данных из числа сохраненных и странице с уведомлением о необходимости дополнить данные.

предварительно выбранногоЕсли пользователь переходит по QR-коду, на другом устройстве ему отображается страница выбора платежного метода, страница указания платежных данных или страница выбора данных из числа сохраненных — в зависимости от того, с какой страницы платежной формы был выполнен переход, и с учетом параметров открытия платежной формы (например,платежного метода). При этом данные, если они были указаны пользователем на первоначальном устройстве, не сохраняются и не отображаются на платежной форме.

Синхронизация экранов на устройствах не поддерживается: на первоначальном устройстве отображается та страница платежной формы, с которой выполнен переход. При этом платежная форма на обоих устройствах остается актуальной до тех пор, пока хотя бы на одном из них пользователем не будет подтверждено выполнение целевого действия. После этого продолжить выполнение целевого действия можно только на том устройстве, на котором было подтверждено выполнение этого действия.

## Особенности и ограничения

Возможность отображения платежной формы на нескольких устройствах поддерживается при соблюдении следующих условий:

- При использовании индивидуального дизайна формы информацию о действиях, необходимых для поддержки возможности ее открытия на нескольких устройствах, следует уточнять у курирующего менеджера Some Platform.Для платежной формы используется типовое оформление от Some Platform.
- При формировании токена возможность не поддерживается.Выполняется одно из следующих целевых действий: оплата, блокировка средств, проверка платежного инструмента или регистрация повторяемой оплаты.
- При выборе платежного метода, для которого не поддерживается данная возможность, на дальнейших страницах платежной формы QR-код не отображается.Возможность поддерживается для выбранного платежного метода (подробнее — у специалистов технической поддержки Some Platform).

## Подключение

Чтобы подключить эту возможность, со стороны мерчанта необходимо обратиться к специалистам технической поддержки (support@example.com Подключение возможно как в рамках проекта мерчанта, так и в рамках работы с конкретным платежным методом. При этом возможность может быть недоступна для отдельных платежных методов, подробнее — у специалистов технической поддержки.
