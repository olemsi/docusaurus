## Особенности проведения платежей

Нужно помнить про некоторые особенности проведения платежей:

- references/ru/currencies/KRW.ditaДля валютысуммы в запросе указываются без дробной части.
- В платежном методе Virtual Accounts при выполнении пользователем оплаты в первый раз ему будет предложено зарегистрироваться в сервисе провайдера. При последующих оплатах передаваемые в запросе данные пользователя должны совпадать с теми, которые пользователь указал при регистрации, в противном случае пользователь не сможет выполнить оплату. Для изменения данных пользователя обратитесь к специалистом технической поддержки Some Platform.
- В платежном методе Virtual Accounts, если пользователь не выполнит необходимые для оплаты действия в течение 20 минут после инициирования оплаты, то платеж будет отклонен по истечении предельного времени проведения платежа.

1. Зарегистрироваться на сайте мерчанта.
2. Зарегистрироваться на сайте провайдера.
3. Для этого пользователь должен перейти на соответствующую страницу сайта провайдера и ввести адрес электронной почты, номер телефона и данные банковского счета. Указанные данные должны принадлежать одному человеку. Процедура KYC потерпит сбой, если, к примеру, указанные номер телефона и банковский счет принадлежат разным людям.Пройти процедуру KYC (Know Your Customer) на сайте провайдера.
4. Для этого пользователь должен:Перейти на соответствующую страницу сайта провайдера.Ввести название магазина мерчанта и щелкнуть кнопку с кодом мерчанта.Ввести адрес электронной почты, который использовался при регистрации на сайте мерчанта, и щелкнуть кнопку подтверждения.Щелкнуть кнопку подтверждения для привязки указанного адреса электронной почты к выбранному коду мерчанта.Связать на сайте провайдера код мерчанта и адрес электронной почты пользователя.
