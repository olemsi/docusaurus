# Выплата через Gate

## Общая информация

Для выплаты через Gate с использованием метода title со стороны веб-сервиса необходимо отправить запрос, содержащий требуемые параметры и подпись, на рабочий URL Some Platform и принять оповещение о результате выплаты. Полная схема проведения выплаты представлена далее.

ru_gate_sts_protocol.ditaИнформация о формате запросов и параметрах инициации выплат методом title через Gate, а также о формате оповещений о результатах выплат приведена далее, общая информация о работе с API см. в разделе.

## Формат запросов

При работе с запросами на выплаты с применением метода title необходимо учитывать следующее:

Вот пример данных из запроса на выплату с использованием метода title:

1. `/v2​/payment​/wallet/bloomzed/payout`Должен использоваться запрос к конечной точке, отправляемый методом POST. Этот запрос относится к группе запросов по проведению платежей с помощью электронных кошельков request-payment-wallet-payout.
2. generalproject_id— идентификатор проекта, полученный от Some Platform при интеграции;payment_id— идентификатор платежа, уникальный в рамках проекта;signatureИспользование подписи к данным— подпись запроса, составленная после определения всех параметров запроса. (Подробнее см. раздел.);— объект, содержащий основные сведения о запросе:paymentamount*за исключениемосновных единицах валюты*ru_currency_codes.dita— сумма выплаты в дробных единицах валюты без десятичной точки и пробеловслучаев, когда у валюты нет дробной части. Если у валюты нет дробных единиц (то есть количество разрядов дробных единиц равно нулю), то в этом параметре нужно указывать сумму в. Подробнее о разрядах дробных единиц у валют см.;currency— код валюты выплаты в формате ISO-4217 alpha-3;— объект, содержащий сведения о выплате:customerip_address— IP-адрес устройства пользователя;id— идентификатор пользователя, уникальный в рамках проекта;first_name— имя пользователя;last_name— фамилия пользователя;email— адрес электронной почты пользователя;— объект, содержащий сведения о пользователе:accountnumber`79031234567`— номер телефона пользователя, на который зарегистрирован электронный кошелек, указывается с кодом страны, без знаков пунктуации и специальных символов (например,).— сведения о счете пользователя:В запросе должны использоваться следующие объекты и параметры:
3. references/ru/currencies/KZT.ditaВалютой платежа может быть только.
4. Дополнительно могут использоваться все параметры, указанные в спецификации.

## Формат оповещений

ru_gate_callbacks.ditaДля оповещений о результатах выплат с применением метода title используется стандартный формат, описание которого представлено в разделе.

`29801100,00 KZT`В следующем примере оповещение свидетельствует о том, что в рамках проектабыла успешно проведена выплата в размере.

В следующем примере выплата была отклонена.
