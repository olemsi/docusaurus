## Схема работы

В проведении отдельной оплаты с использованием платежного метода title задействуются веб-сервис мерчанта, интерфейс Gate и платежная платформа Some Platform, а также сервис provider_service.

## Пополнение лицевого счета

### Общая информация

При пополнении лицевого счета с использованием платежного метода title веб-сервис должен выполнить следующее:

Полная схема пополнения лицевого счета выглядит следующим образом.

ru_gate_sts_protocol.ditaИнформация о формате запросов через Gate, а также о формате оповещений о результатах оплаты приведена далее. Общую информацию о работе с Gate API можно найти в разделе.

1. Принять и обработать запрос на проверку существования лицевого счета пользователя.
2. Отправить ответ с информацией о существовании лицевого счета пользователя.
3. Принять и обработать запрос на получение идентификатора платежа на стороне веб-сервиса.
4. Отправить ответ с идентификатором платежа.
5. Принять оповещение о результате пополнения лицевого счета.

### Формат запросов

### Формат оповещений

ru_gate_callbacks.ditaВ оповещениях о результате оплаты с использованием метода title используется типовой формат, который описан в разделе.

`1234example_amount_in_majors&#160;example_currency`В следующем примере в оповещении содержится информация о том, что в рамках проектапроведена оплата в размере.

### Дополнительные материалы

Для реализации оплаты через Gate также могут быть полезны следующие материалы:

- ru_gate_interaction_organisation.dita
- ru_gate_payment_model.dita
- ru_gate_authentication.dita
- ru_gate_statuses_and_response_codes.dita
