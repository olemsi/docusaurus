# Схема и статусы разовой оплаты в одну стадию

## Общая информация

*Разовая оплата в одну стадиюразовая одностадийная оплата*, или, — это тип платежа, в рамках которого на основании одного исходного запроса осуществляется один (разовый) перевод денежных средств от пользователя к мерчанту. Это базовый вариант для проведения оплат — с незамедлительным разовым списанием средств (например, для расчета за совершенную покупку).

## Схема проведения

`salesale`Чтобы инициировать оплату в одну стадию, следует отправить в платформу запрос категории. Для выполнения такого запроса в платформе формируется операция, результатом выполнения которой является списание средств со счета пользователя.

При проведении оплаты в одну стадию дополнительно могут требоваться и другие запросы:

*возврат средств*`refund`Если для использованного платежного метода поддерживается проведение возвратов, то после проведения разовой оплаты в одну стадию по этой оплате можно выполнитьпользователю. Чтобы инициировать возврат, следует отправить в платформу запрос категории. Для выполнения такого запроса в платформе используется одна из следующих операций:

ru_gate_payment_sale.ditaru_pm_intro.dita, а об оплате с применением альтернативных платежных инструментов — в разделеДалее в рамках данного раздела представлена информация о возможных статусах разовой одностадийной оплаты и связанных с ней операций. Более подробную информацию о проведении разовой одностадийной оплаты с использованием платежных карт можно найти в разделе.

При проведении разовой одностадийной оплаты могут использоваться следующие статусы.

| `error` | Проведение платежа не инициировано из-за ошибки, возникшей при проверке принятого запроса | *Конечное состояние. Допускается повторная отправка запроса с тем же идентификатором платежа и повторная попытка проведения этого платежа* |
| --- | --- | --- |
| `processing` | Платеж проводится | *Промежуточное состояние* |
| `awaiting 3ds result` | `3ds_resultdecline`Проведение платежа приостановлено до получения запроса на продолжение с учетом результата аутентификации 3ds —. Если такой запрос не получен в течение установленного времени, то платеж переводится в статус. Как правило, время ожидания такого запроса составляет 30 минут, но может варьироваться в зависимости от используемого провайдера. Для получения более подробной информации о времени ожидания следует обращаться к специалистам технической поддержки — support@example.com | *Промежуточное состояние* |
| `awaiting merchant auth` | `merchant_authfinishtype`Проведение платежа приостановлено до получения запроса на завершение аутентификации пользователя в платежной системе по инициативе мерчанта —со значениемдля параметра | *Промежуточное состояние* |
| `awaiting redirect result` | `successdecline`Проведение платежа приостановлено до получения уведомления с результатом со стороны платежной системы. В зависимости от результата на стороне платежной системы платеж переводится в статусили.`awaiting redirect resultawaiting customer action`В рамках проведения одного платежа может использоватьсялибо, но не оба этих статуса | *Промежуточное состояние* |
| `awaiting customer action` | (в соответствии со спецификой платежного метода)`successdecline`Проведение платежа приостановлено до выполнения необходимых действий пользователем со стороны платежной системы. В зависимости от результата этих действий платеж переводится в статусили статус.`awaiting customer actionawaiting redirect result`В рамках проведения одного платежа может использоватьсялибо, но не оба этих статуса | *Промежуточное состояние* |
| `awaiting clarification` | `clarificationdecline`ru_gate_clarification.ditaПроведение платежа приостановлено до получения запроса на продолжение с учетом требуемой дополнительной информации —(подробнее — в разделе). Если такой запрос не получен в течение 30 минут, платеж переводится в статус | *Промежуточное состояние* |
| `awaiting customer` | `successdecline`ru_pp_try_again.ditaПроведение платежа приостановлено до получения результата повторных попыток со стороны пользователя. При успешной повторной попытке платеж переводится в статус, а при истечении числа безуспешных попыток — в статус(подробнее — в разделе) | *Промежуточное состояние* |
| `decline` | Платеж отклонен | *Конечное состояние* |
| `success` | Платеж проведен | *Конечное состояние. Дополнительно допускается проведение возврата* |
| `partially paid` | `sale`Платеж с разбиением суммы частично проведен одной из операций | *Конечное состояние. Дополнительно допускается проведение возврата* |
| `partially refunded` | Сумма платежа частично возвращена | *Конечное состояние. Дополнительно допускается отмена возврата* |
| `refunded` | Сумма платежа полностью возвращена после закрытия бизнес-дня, в котором он был проведен. Осуществлен один полный возврат суммы платежа или несколько частичных, в совокупности составляющих исходную сумму | *Конечное состояние. Дополнительно допускается отмена возврата* |
| `partially reversed` | Сумма платежа частично возвращена | *Конечное состояние* |
| `reversed` | Сумма платежа полностью возвращена до закрытия бизнес-дня, в котором он был проведен | *Конечное состояние* |
