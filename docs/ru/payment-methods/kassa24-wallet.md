## Оплата через Gate

### Общая информация

Для оплаты через Gate с использованием метода title со стороны веб-сервиса необходимо:

Полная схема проведения оплаты выглядит следующим образом.

ru_gate_sts_protocol.ditaИнформация о формате запросов и параметрах инициирования оплаты методом title через Gate, а также о формате оповещений о результатах оплаты приведена далее. Общую информацию о работе с Gate API можно найти в разделе.

1. Отправить запрос, содержащий обязательные параметры и подпись, на URL-адрес Some Platform.
2. Получить оповещение с данными для отображения платежной инструкции и отобразить пользователю инструкцию для оплаты.
3. Принять оповещение от платежной платформы Some Platform о результате оплаты.

### Формат запросов

При работе с запросами на оплату с использованием метода title необходимо учитывать следующее:

Вот пример данных из запроса на оплату с использованием метода title:

1. `/v2/payment/wallet/kassa24/sale`Должен использоваться POST-запрос к конечной точке. Эта конечная точка относится к группе конечных точек request-payment-wallet-sale.
2. generalproject_id— идентификатор проекта, полученный от Some Platform при интеграции;payment_id— идентификатор платежа, уникальный в рамках проекта;signatureИспользование подписи к данным— подпись запроса, составленная после указания целевых параметров (подробнее — в разделе);— объект, содержащий основные сведения о запросе:customerid— идентификатор пользователя, уникальный в рамках проекта;ip_address— IP-адрес устройства пользователя;first_name— имя пользователя;last_name— фамилия пользователя;email— адрес электронной почты пользователя;— объект, содержащий сведения о пользователе:paymentamount*за исключениемосновных единицах валюты*ru_currency_codes.dita— сумма оплаты в дробных единицах валюты без десятичной точки и пробеловслучаев, когда у валюты нет дробной части. Если у валюты нет дробных единиц (то есть количество разрядов дробных единиц равно нулю), то в этом параметре нужно указывать сумму в. Подробнее о разрядах дробных единиц у валют см.;currency— код валюты платежа в формате ISO-4217 alpha-3;— объект, содержащий сведения о платеже:В запросе должны использоваться следующие объекты и параметры:
3. phonecustomerru_gate_clarification.ditaДополнительно надо передать номер телефона пользователя в параметреобъекта. Если этого параметра не будет в запросе, платежная платформа запросит его дополнительно.  Подробнее о запросе дополнительных параметров см..
4. references/ru/currencies/KZT.ditaВалютой платежа может быть только.
5. return_urlreturn_urlsuccess— URL-адрес, на который нужно перенаправить пользователя в случае успешного завершения оплаты;decline— URL-адрес, на который нужно перенаправить пользователя в случае отклонения оплаты;returnsuccessdecline— URL-адрес, куда нужно перенаправить пользователя в случае прерывания оплаты пользователем до ее завершения. Этот же адрес используется, если не заданы параметрыи.— объект с информацией, куда следует перенаправлять пользователя при прерывании или завершении оплаты:Следующие параметры с адресами возврата пользователя необязательны, но их настоятельно рекомендуется передавать в запросе, чтобы сделать процесс оплаты максимально удобным для пользователя. Если не задан ни один из параметров объекта, то Gate по умолчанию перенаправляет пользователя по URL-адресу, указанному в вашем проекте в Some Platform.
6. Дополнительно в запросе могут использоваться любые другие (необязательные) объекты и параметры, указанные в спецификации.

### Данные для перенаправления пользователей

После получения и обработки запроса на оплату платежная платформа направляет в ваш веб-сервис оповещение (callback) с данными для перенаправления пользователя в сервис provider_service.

redirect_dataЧтобы перенаправить пользователя, используйте значения следующих параметров, находящихся в объекте:

Вот пример фрагмента оповещения, содержащего данные для перенаправления пользователя.

- url— адрес для перенаправления пользователя;
- body— данные для отправки в теле запроса;
- method— метод запроса HTML-страницы сайта (например, POST или GET);
- encrypted— это служебный параметр. Игнорируйте содержащиеся в нем данные.

```json
"redirect_data": {
    "method": "POST",
    "body": {
        "example_parameter_1": "value_1",
        "example_parameter_2": "value_2"
    },
    "encrypted": [],
    "url": "https://example.com/redirect"
}
```
