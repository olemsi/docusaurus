# Глоссарий

## API (Application Programming Interface)

Набор правил, согласно которым программные системы взаимодействуют друг с другом и обмениваются данными. Для проведения платежей с использованием платежной платформы Some Platform вам доступны следующие API:

- Payment Widget API — программный интерфейс, с помощью которого можно открыть. В Some Platform оплата интегрируется только с использованием страницы оплаты;страницу оплаты Payment Widget
- Gate API — программный интерфейс, с помощью которого вы можете проводить выплату пользователям. В Some Platform выплата может быть инициирована с использованиеми.

## Dashboard

Пользовательский интерфейс, с помощью которого вы можете просматривать и анализировать информацию по проведенным платежам, инициировать оплату, возврат и выплату, контролировать и предотвращать попытки мошенничества и выполнять иные действия.

## Gate

Компонент платежной платформы Some Platform, обеспечивающий безопасное и эффективное взаимодействие междуи платежной платформой. В Some Platform вы можете использовать Gate для проведения выплаты пользователям вашей системы. Взаимодействие с компонентом Gate осуществляется посредством Gate API.вашей системой

## Payment Widget

Пользовательский интерфейс, представляющий собой страницу оплаты, с помощью котороймогут совершать оплату. Взаимодействие со страницей оплаты осуществляется посредством Payment Widget API.с использованием различныхплатежных методовпользователи

## Ваша система

Сайт, мобильное приложение или другой сервис, где вы предоставляете товары или услуги.пользователям

## Оповещение (callback)

Техническое сообщение, которое платежная платформа отправляет на заданный URL-адрес вашей системы, чтобы сообщить определенную информацию, например о результате выполнения операции. Оповещение представляет собой HTTP-POST-запрос определенной структуры, но вы можете настроить структуру, параметры адресации и случаи, в которых отправляются оповещения, в соответствии с вашими потребностями.

## Основная валюта

Валюта, заданная в качестве основной для платежного метода, настроенного в вашем. В этой валюте будет обработана операция, если в запросе на проведение платежа вы передадите валюту, отличную от валюты, допустимой для платежного метода.проекте

## Платеж

Комплекс действий по выполнению запроса от вашей системы на перевод денежных средств между вашей системой и. В настоящий момент в Some Platform поддерживаются следующие типы платежей:пользователем

На стороне вашей системы каждому платежу присваивается свой идентификатор платежа. Убедитесь, что этот идентификатор не повторяется в рамках одного вашегов Some Platform.проекта

- — перевод средств от пользователя в вашу систему;**оплата (purchase)**
- — перевод средств от вашей системы к пользователю.**выплата (payout)**

## Платежный метод

Способ, который выбирает пользователь для совершения платежа. Платежные методы могут относиться к разными регионам, поддерживать определенные валюты и типы платежей, а также иметь различные пользовательские сценарии, особенности и ограничения.типам

Примеры платежных методов, поддерживаемых в Some Platform — Bank Transfers, DANA, SPEI, UPI. Подробнее о платежных методах см. раздел.ru_qck_pm_countries.dita

## Подпись

Строка данных, полученная заданным способом из набора подписываемых данных и. Подпись используется для обеспечения подлинности и целостности данных, передаваемых в технических запросах имежду вашей системой и Some Platform. Подробнее о формировании подписи, см. раздел.секретного ключаоповещениях

## Пользователь

Лицо, которое оплачивает товары или услуги в.вашей системе

## Проект

Сущность, созданная внутри платежной платформы и закрепленная за. В Some Platform у каждого проекта есть свой наборпараметров, влияющих на проведение платежей и порядок взаимодействия платежной платформы с вашей системой (в частности, при работе с оповещениями).платежных методов ивашей системой

Для каждого проекта вам выдается отдельный идентификатор проекта, который вы должны использовать при взаимодействии с платежной платформой.

## Секретный ключ

Ключ симметричного шифрования данных. Секретный ключ уникален для каждого проекта и используется для формирования и проверки корректности цифровой.подписи

## Тип платежного метода

Категория, которая включает в себя один или несколько, одинаковых по способу проведения платежей., например: электронные кошельки, банковские переводы, оплата наличными, оплата с использованием ваучеров и другиеплатежных методов
