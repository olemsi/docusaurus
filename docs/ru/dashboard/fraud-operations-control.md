# Контроль мошеннических операций

Dashboard позволяет отслеживать информацию о попытках и фактах мошенничества, выявленных на стороне Some Platform и платежных систем. Для этого можно использовать различные инструменты:

подробнееВ работе с указанными реестрами доступны типовые инструменты фильтрации (), а также карточки с детальной информацией об отдельных платежах и относящихся к ним операциях (для открытия карточки достаточно щелкнуть строку платежа в реестре).

**Риски**`RisksMerchant Admin`Для работы с детальной информацией о мошенничестве в разделеи карточках платежей требуется право на управление рисками, по умолчанию доступное учетным записям с ролямии, кроме того для работы с отчетами требуется отдельное право на управление отчетами.

При работе с реестрами платежей и мошеннических действий следует учитывать ряд особенностей:

Для контроля информации об интересующих операциях достаточно следующих действий:

1. **ПлатежиРиски**Перейти в нужный раздел:или.
2. `decline402fraud`RCSВ реестре платежей операции, отклоненные на стороне Some Platform из-за их высокого риска, можно найти по статусу платежаи служебному коду (к таким кодам относятся коди коды блока), а операции, признанные мошенническими другими сторонами, можно найти по индикатору(в том числе с помощью фильтрации).Найти в реестре записи о требуемых операциях, используя инструменты фильтрации, если это необходимо.
3. **Мошеннические платежи**В реестре мошеннических действий и в блоке, расположенном в детальной карточке платежа, отображается информация о мошенничестве в рамках конкретной операции.Проверить интересующую информацию, непосредственно в реестре или в карточках платежей.
