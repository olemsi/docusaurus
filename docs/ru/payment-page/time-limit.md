# Ограничение времени работы с Payment Widget

## Общая информация

При использовании Payment Widget поддерживается возможность указывать срок действия платежной формы, до истечения которого пользователь может работать с ней для выполнения целевого действия. В случае, если этот срок задан, то до подтверждения пользователем выполнения целевого действия на страницах Payment Widget дополнительно отображается оставшееся время работы с платежной формой.

Ограничение срока действия Payment Widget позволяет контролировать предоставление пользователям услуг с привязкой ко времени и может быть актуально, например, при продаже билетов или распродаже товаров.

ru_pp_clarification.ditaПри проведении платежей может потребоваться выполнение процедуры дополнения информации о платеже. В таком случае после подтверждения пользователем целевого действия отсчет времени работы с платежной формой останавливается, а при отображении страницы ввода дополнительных данных на Payment Widget отображается новый отсчет времени, установленный в платежной платформе по умолчанию и составляющий 30 минут с момента отображения этой страницы. Подробная информация о процедуре дополнения информации о платеже представлена в разделе.

## Пользовательский сценарий

Базовый пользовательский сценарий выполнения оплаты при ограничении времени работы с Payment Widget можно представить следующим образом:

1. `mm:ss`Пользователь в веб-сервисе мерчанта подтверждает готовность оплатить свой заказ и перенаправляется на Payment Widget. При этом на Payment Widget отображается оставшееся время работы в формате.
2. Если пользователь не подтверждает выполнение целевого действия до истечения срока действия, указанного в запросе на проведение платежа, ему отображается страница с уведомлением об истечении этого срока.Пользователь указывает реквизиты платежного инструмента, подтверждает готовность оплатить заказ и ожидает результат. После этого выполнение целевого действия считается подтвержденным и время работы с Payment Widget на следующих страницах не отображается.

## Формат запроса

ru_pp_interaction_organisation.dita`best_beforeYYYY-MM-DDThh:mm:ss±hh:mm`Формат запроса на открытие Payment Widget для выполнения оплаты с указанием срока действия платежной формы, в течение которого пользователь может работать с ней для выполнения целевого действия, соответствует описанному в разделе. Для указания срока действия платежной формы в этом запросе необходимо дополнительно передать параметрсо значением в формате.

```json
{  
    // обязательные параметры для выполнения оплаты
   "project_id": "42"
   "payment_id": "7654321777"
   "payment_currency": "USD"
   "payment_amount": "131970"
   "customer_id": "customer_12"
   "signature": "TSzdE5rJZaA9TYAKoGpfXriFf82MxF..."

    //срок использования Payment Widget для выполнения целевого действия — до 10:15:30 12 апреля 2021 года, GMT+3
   "best_before": "2021-04-12T10:15:30+03:00"  
 
}
```

::: tip
*pp_host*Здесь вместо <> надо указать доменное имя Payment Widget, которое следует узнавать у своего курирующего менеджера или в службе поддержки.
:::
